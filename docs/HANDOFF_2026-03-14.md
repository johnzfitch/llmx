## Current behavior
- JS/TS/TSX still use the existing legacy tree-sitter implementation through the registry.
- Rust and other newly supported code languages no longer fall straight to text chunking if a grammar is available.
- Generic structural chunks now populate at least:
  - `ast_kind`
  - `qualified_name`
  - `visibility` for simple Rust `pub` cases
  - `resolution_tier = GenericTreeSitter` through the top-level `chunk_file` entrypoint
- There are unit tests covering generic Rust function and impl-method chunking.

## Most important unfinished work

### 1. Wire `symbol_id` into emitted structural chunks
Start in:
- `ingestor-core/src/chunk/generic.rs`
- `ingestor-core/src/chunk/symbol_id.rs`

Goal:
- set `chunk.symbol_id`
- keep `chunk.symbol_tail`

Current gap:
- `symbol_id.rs` exists but is not used
- generic chunks still emit `symbol_id = None`

### 2. Build the real Rust adapter
Spec target:
- Rust should not stay on the generic fallback long-term

Start from:
- `ingestor-core/src/chunk/generic.rs`
- create new file `ingestor-core/src/chunk/parsers/rust.rs` or equivalent after deciding final layout

Needed capabilities:
- proper `fn`, `struct`, `enum`, `trait`, `mod`, `impl`, methods
- `use` imports
- call extraction
- type refs
- doc summaries
- robust module / impl qualification

Current limitation:
- generic adapter is only definition-oriented
- no Rust imports/calls/type edges yet

### 3. Finish the module split toward spec layout
The code is only partially split.

Still missing from spec direction:
- `query_loader.rs`
- `registry` with true three-tier dispatch semantics
- parser-tier specific modules rather than generic + legacy only
- final `chunk/mod.rs` style layout if desired

### 4. Decide what to do with `ResolutionTier` at ingest stamping time
Current behavior:
- `chunk::chunk_file()` now sets `chunk.resolution_tier` from adapter result
- later ingest path in `ingestor-core/src/lib.rs` still stamps file-level resolution based mostly on language detection

This is acceptable for current generic parsing, but it will become wrong once stack-graph and query-pack tiers land.

### 5. Query packs and stack graphs are still not started
Nothing yet for:
- `.scm` query loading
- stack-graphs integration
- incremental parse cache
